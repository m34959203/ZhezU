## 1. Стратегический обзор

### 1.1 Видение

ZhezU -- единая цифровая экосистема Жезказганского университета, сопровождающая пользователя от подачи заявки абитуриентом до участия в алумни-сообществе. Платформа должна стать ключевым инструментом конкурентоспособности университета в условиях прогнозируемого демографического спада (~15 % к 2026 г.), когда 47 % абитуриентов используют сайт вуза как основной источник информации при выборе.

Два стратегических дифференциатора, отсутствующих у конкурентов:
- **Talapker** -- система геймификации жизненного цикла студента (20 уровней, 28 квестов, 12 значков, лидерборд);
- **AI-Центр** -- полноценный раздел навигации верхнего уровня (по модели MIT Innovation), включающий AI-проекты, AI-агентов, AI-приложения, AI-инструменты и AI-лабораторию.

### 1.2 Текущее состояние проекта (обновлено 26.02.2026)

По результатам аудита (февраль 2026) текущий сайт zhezu.edu.kz оценен на **3.6 из 10** и скомпрометирован. Параллельно создан новый фронтенд на современном стеке. **Проект значительно продвинулся с момента первоначального аудита.**

**Что реализовано (актуальное состояние):**

Критический путь определяет минимально возможную длительность проекта. Любая задержка на критическом пути сдвигает дату запуска.

                              |                          |

                    [Frontend CMS Integration (2.3.3)]

                    [QA Cycle (4.1.1)] --> [Bugfix (4.2.1)] --> [DNS Switch (4.2.5)]

                                                                       |

                                                             [AI Center (6.1.x)]

### 9.2 Параллельные потоки

Следующие потоки работ могут выполняться параллельно:

```
Поток A (Frontend):      Компоненты --> MVP Pages --> QA --> Portal UI --> AI Center UI
Поток B (Backend):       CMS Setup  --> API Layer --> Auth  --> Integrations --> AI ML
Поток C (DevOps):        CI/CD --> Docker --> Staging --> Production --> Monitoring
Поток D (UX/Content):    Audit --> Prototypes --> Content --> Localization --> User Testing
Поток E (QA):            Test setup --> Component tests --> E2E --> Security --> Regression
```

### 9.4 Блокеры: что задержит проект

| Блокировка DNS-переключения | Запуск MVP | Низкая | Заранее согласовать с IT-отделом университета и регистратором |
