## 1. Введение

Настоящий документ описывает концепцию проекта **ZhezU** — новой цифровой платформы для Жезказганского университета имени О.А. Байконурова. Проект предусматривает полное обновление официального сайта университета по адресу https://zhezu.edu.kz/ с созданием современного, мультиязычного, высокопроизводительного веб-приложения на основе модели **«hub-and-spoke»** (центральный портал + факультетские микросайты).

Документ объединяет и систематизирует требования из:
- **ТЗ_Веб-сайт_университета_v1.0** — техническое задание с 82 функциональными требованиями
- **Концепция_проекта_сайта_университета v1.0** — стратегическое видение и архитектура
- **Анализ текущего кода и прототипов** — аудит реализованного функционала

## 2. Предпосылки и обоснование проекта

### 2.1 Текущее состояние сайта

По результатам аудита (февраль 2026) текущий сайт zhezu.edu.kz получил общую оценку **3.6 из 10** по совокупности критериев:

| Критерий | Оценка |
|----------|:------:|
| Визуальная привлекательность | 4/10 |
| Навигация и архитектура | 3/10 |
| Адаптивность (responsive) | 5/10 |
| Типографика и читаемость | 6/10 |
| Цветовая система | 5/10 |
| Контент и UX-копирайтинг | 3/10 |
| Доступность (a11y) | 3/10 |
| Скорость загрузки | 4/10 |
| SEO | 2/10 |
| Безопасность | 1/10 |

### 2.2 Ключевые проблемы

1. **Безопасность** — сайт был скомпрометирован, на страницах размещался мошеннический контент
2. **Устаревший дизайн** — не соответствует стандартам 2025-2026 для сайтов высших учебных заведений
3. **Слабая навигация** — отсутствует мегаменю, поиск, быстрые пути для разных аудиторий
4. **Нет персонализации** — контент не разделён по типам пользователей (абитуриент, студент, преподаватель)
5. **Нет интерактивности** — отсутствуют чат-бот, фильтры программ, калькуляторы, виртуальный тур
6. **Нет тёмной темы** — стандартное ожидание аудитории поколения Z
7. **Ограниченная мобильная версия** — базовая адаптивность без полноценного мобильного UX
8. **Нет CMS** — весь контент захардкожен, обновление требует разработчика
9. **Нет аналитики** — отсутствует система мониторинга, логирования, Sentry

### 2.3 Контекст рынка образования

По данным исследований, **47% студентов** используют сайт университета как основной источник информации при выборе ВУЗа. В условиях прогнозируемого демографического спада (~15% к 2026 году) качество сайта становится критическим фактором конкурентоспособности университета в привлечении абитуриентов.

## 3. Цели и задачи проекта

Создать единую цифровую платформу Жезказганского университета, которая:
- Повысит привлекательность университета для абитуриентов
- Обеспечит современные цифровые сервисы для студентов и преподавателей
- Станет инструментом продвижения университета на рынке образования Казахстана
- Послужит витриной AI-Центра университета

## 5. Информационная архитектура

### 5.1 Модель Hub-and-Spoke

```
                        ┌──────────────────────┐
                        │   ЦЕНТРАЛЬНЫЙ ПОРТАЛ  │
                        │     zhezu.edu.kz      │
                        │                       │
                        │ ▪ Главная страница    │
                        │ ▪ Университет         │
                        │ ▪ Поступление         │
                        │ ▪ Образование         │
                        │ ▪ Наука               │
                        │ ▪ AI-Центр            │
                        │ ▪ Жизнь в университете│
                        │ ▪ Новости и события   │
                        │ ▪ Контакты            │
                        └─────────┬─────────────┘
                                  │
            ┌─────────┬───────────┼───────────┬──────────┐
            ▼         ▼           ▼           ▼          ▼
    ┌───────────┐ ┌───────────┐ ┌───────────┐ ┌───────────┐ ┌───────────┐
    │ Кафедра   │ │ Кафедра   │ │ Кафедра   │ │ Кафедра   │ │ Кафедра   │
    │ Горное    │ │ Педагогика│ │ История   │ │ Электро-  │ │ Тех.      │
    │ дело      │ │ и филол.  │ │ и право   │ │ энергетика│ │ машины    │
    │           │ │           │ │           │ │           │ │           │
    │ /dept/    │ │ /dept/    │ │ /dept/    │ │ /dept/    │ │ /dept/    │
    │ mining    │ │ pedagogy  │ │ history   │ │ energy    │ │ techeng   │
    └───────────┘ └───────────┘ └───────────┘ └───────────┘ └───────────┘
```

Центральный портал содержит общеуниверситетский контент, а каждая кафедра получает свой микросайт (spoke) с единым шаблоном, но индивидуальным контентом — программы, преподаватели, публикации, новости.

### 5.2 Трёхуровневая навигация

| Уровень | Содержание | Расположение |

| **Audience Links** | Абитуриентам, Студентам, Преподавателям, Работодателям | Под основной навигацией |

**6 мегаменю первичной навигации:**

1. **Университет** — О вузе, руководство, история, партнёры, аккредитация, документы

3. **Образование** — Бакалавриат, магистратура, кафедры, расписание, библиотека
4. **Наука** — Публикации, конференции, лаборатории, гранты, Scopus/WoS
5. **AI-Центр** — Проекты, Агенты, Приложения, Инструменты, Лаборатория
6. **Жизнь** — Новости, события, спорт, общежития, студенческие организации

## 6. Концепция продукта

### 6.1 Видение

**ZhezU** — единая цифровая экосистема университета, в которой абитуриент начинает свой путь с подачи заявки и проходит все этапы студенческой жизни до трудоустройства и участия в алумни-сообществе. Платформа объединяет 8 функциональных модулей в рамках модели hub-and-spoke.

### 6.2 Ключевые принципы

| Принцип | Описание |
|---------|----------|
| **Mobile-first** | Дизайн в первую очередь для мобильных — 80%+ целевой аудитории (абитуриенты) на смартфонах |
| **Три языка** | Казахский (қазақша) — государственный, русский — для большинства, английский — для международных |
| **Atomic Design** | Атомы → Молекулы → Организмы → Шаблоны → Страницы для масштабируемой компонентной архитектуры |
| **Hub-and-Spoke** | Центральный портал + микросайты кафедр с единым шаблоном |
| **Доступность** | WCAG 2.2 AA — контраст, клавиатура, скринридеры, prefers-reduced-motion |
| **Производительность** | LCP ≤ 2.5s, FID ≤ 100ms, CLS ≤ 0.1, TTFB ≤ 200ms, Lighthouse ≥ 90 |
| **Безопасность** | WAF, CSP, мониторинг целостности, pen-testing, SLA 99.5% |
| **Контент-менеджмент** | Headless CMS для управления контентом без разработчика |

- **Значки** — 12 видов достижений разных степеней редкости
- **Лидерборд** — рейтинг среди сверстников

| Lighthouse Accessibility | ≥ 90 | Не измерено |

- Все изображения с alt-атрибутами
- Все формы со связанными `<label>` и `aria-describedby` для ошибок
- Все интерактивные элементы доступны с клавиатуры
- Контраст текста: 4.5:1 (обычный) / 3:1 (крупный)
- Skip-links для навигации
- `aria-current="page"` для активных пунктов меню
- `prefers-reduced-motion` для анимаций
- Focus-visible стили для всех интерактивных элементов
- Focus-trap в модальных окнах
- Live regions (`aria-live`) для динамического контента

- SSR/SSG для всех публичных страниц
- Мета-теги на каждом языке (title ≤ 60, description ≤ 160)

- Sitemap.xml и robots.txt
- Open Graph и Twitter Card
- Canonical URLs

- HTTPS повсеместно
- Content Security Policy (CSP)
- Web Application Firewall (WAF) — Cloudflare
- Мониторинг целостности файлов
