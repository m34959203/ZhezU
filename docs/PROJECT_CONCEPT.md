**Версия:** 1.0
**Дата:** 8 февраля 2026
**Заказчик:** АО «Жезказганский университет имени О.А. Байконурова»
**Целевой сайт:** https://zhezu.edu.kz/

---

## 1. Введение

Настоящий документ описывает концепцию проекта **ZhezU** — новой цифровой платформы для Жезказганского университета имени О.А. Байконурова. Проект предусматривает полное обновление официального сайта университета по адресу https://zhezu.edu.kz/ с созданием современного, мультиязычного, высокопроизводительного веб-приложения.

---

## 2. Предпосылки и обоснование проекта

### 2.1 Текущее состояние сайта

По результатам аудита (февраль 2026) текущий сайт zhezu.edu.kz получил общую оценку **3.6 из 10** по совокупности критериев:

| Критерий | Оценка |
|----------|:------:|
| Визуальная привлекательность | 4/10 |
| Навигация и архитектура | 3/10 |
| Адаптивность (responsive) | 5/10 |
| Типографика и читаемость | 6/10 |
| Цветовая система | 5/10 |
| Контент и UX-копирайтинг | 3/10 |
| Доступность (a11y) | 3/10 |
| Скорость загрузки | 4/10 |
| SEO | 2/10 |
| Безопасность | 1/10 |

### 2.2 Ключевые проблемы

1. **Безопасность** — сайт был скомпрометирован, на страницах размещался мошеннический контент
2. **Устаревший дизайн** — не соответствует стандартам 2025-2026 для сайтов высших учебных заведений
3. **Слабая навигация** — отсутствует мегаменю, поиск, быстрые пути для разных аудиторий
4. **Нет персонализации** — контент не разделён по типам пользователей (абитуриент, студент, преподаватель)
5. **Нет интерактивности** — отсутствуют чат-бот, фильтры программ, калькуляторы, виртуальный тур
6. **Нет тёмной темы** — стандартное ожидание аудитории поколения Z
7. **Ограниченная мобильная версия** — базовая адаптивность без полноценного мобильного UX

### 2.3 Контекст рынка образования

По данным исследований, **47% студентов** используют сайт университета как основной источник информации при выборе ВУЗа. В условиях прогнозируемого демографического спада (~15% к 2026 году) качество сайта становится критическим фактором конкурентоспособности университета в привлечении абитуриентов.

## 3. Цели и задачи проекта

### 3.1 Стратегическая цель

Создать единую цифровую платформу Жезказганского университета, которая:
- Повысит привлекательность университета для абитуриентов
- Обеспечит современные цифровые сервисы для студентов и преподавателей
- Станет инструментом продвижения университета на рынке образования Казахстана

### 3.2 Задачи

| # | Задача | Приоритет |

## 5. Концепция продукта

### 5.1 Видение

**ZhezU** — единая цифровая экосистема университета, в которой абитуриент начинает свой путь с подачи заявки и проходит все этапы студенческой жизни до трудоустройства и участия в алумни-сообществе.

### 5.2 Ключевые принципы

| Принцип | Описание |
|---------|----------|
| **Mobile-first** | Дизайн в первую очередь для мобильных устройств — 80%+ целевой аудитории (абитуриенты) используют смартфоны |
| **Три языка** | Казахский (қазақша) — государственный язык, русский — для большинства пользователей, английский — для международных студентов и SEO |
| **Геймификация** | Система уровней, квестов и значков для мотивации абитуриентов (система «Talapker») |
| **Доступность** | Соответствие WCAG 2.1 AA — контраст, клавиатурная навигация, поддержка скринридеров |
| **Производительность** | Core Web Vitals: LCP < 2.5s, INP < 200ms, CLS < 0.1 |
| **Безопасность** | WAF, мониторинг целостности, регулярные обновления, защита API-ключей |

### 5.4 Жизненный цикл пользователя (Talapker)

Ключевая концепция — сопровождение пользователя на всём пути взаимодействия с университетом:

```
  АБИТУРИЕНТ          СТУДЕНТ 1-2          СТУДЕНТ 3-4         ВЫПУСКНИК
  (Talapker)            курс                  курс              (Alumni)
┌─────────────┐   ┌─────────────┐   ┌──────────────┐   ┌──────────────┐
│ Регистрация │   │ Адаптация   │   │ Научная      │   │ Алумни-      │
│ Документы   │──▶│ Первая      │──▶│  работа      │──▶│  сообщество  │
│ ЕНТ         │   │  сессия     │   │ Лидерство    │   │ Менторство   │
│ Виртуальный │   │ Элективы    │   │ Сертификаты  │   │ Карьерный    │
│  тур        │   │ Стажировка  │   │ Диплом       │   │  рост        │
│ Подача      │   │ CV          │   │ Трудо-       │   │              │
│  заявки     │   │             │   │  устройство  │   │              │
└─────────────┘   └─────────────┘   └──────────────┘   └──────────────┘
 Уровни 1-6        Уровни 7-12       Уровни 13-18      Уровни 19-20
 XP: 0-3000        XP: 3000-12000    XP: 12000-30000   XP: 30000+
```

Каждый этап включает:
- **Квесты** — конкретные действия с начислением XP (18 для абитуриентов, 14 для студентов)
- **Уровни** — 20 уровней от «Жаңа Талапкер» до «ZhezU Legend»
- **Значки** — 12 видов достижений разных степеней редкости
- **Лидерборд** — рейтинг среди сверстников

## 6. Технологическая архитектура

### 6.1 Стек технологий

| Слой | Технология | Обоснование |
|------|-----------|------------|
| **Фреймворк** | Next.js 16 (App Router) | SSR/SSG для SEO, встроенная поддержка i18n, React Server Components |
| **Язык** | TypeScript 5 | Строгая типизация, масштабируемость, безопасность |
| **UI** | React 19 | Современная компонентная модель, серверные компоненты |
| **Стили** | Tailwind CSS 4 | Build-time компиляция, соответствие прототипам, дизайн-токены |
| **i18n** | next-intl 4 | Маршрутизация (/kk, /ru, /en), серверный и клиентский перевод |
| **Темы** | next-themes | Тёмная/светлая тема с системным определением |
| **Анимации** | Framer Motion 12 | Микроанимации, переходы между страницами |
| **Состояние** | Zustand 5 | Лёгкий state management для клиентских данных |
| **Формы** | React Hook Form + Zod | Валидация форм на клиенте и сервере |
| **Иконки** | Lucide React | SVG-иконки, tree-shakeable |
| **PWA** | next-pwa / Serwist | Офлайн-доступ, установка на мобильные |
| **Деплой** | Docker + Railway / Vercel | Контейнеризация, автоматический деплой |

## 10. Критерии качества

### 10.1 Производительность (Core Web Vitals)

| Метрика | Целевое значение |
|---------|:---------------:|
| LCP (Largest Contentful Paint) | < 2.5 сек |
| INP (Interaction to Next Paint) | < 200 мс |
| CLS (Cumulative Layout Shift) | < 0.1 |
| Lighthouse Performance | > 90 |

### 10.2 Доступность (WCAG 2.1 AA)

- Все изображения с alt-атрибутами
- Все формы со связанными `<label>`
- Все интерактивные элементы доступны с клавиатуры
- Контраст текста: 4.5:1 (обычный) / 3:1 (крупный)
- Skip-links для навигации
- `aria-current="page"` для активных пунктов меню
- `prefers-reduced-motion` для анимаций

### 10.3 SEO

- SSR/SSG для всех публичных страниц
- Мета-теги на каждом языке
- `hreflang` для мультиязычности
- Структурированные данные (JSON-LD) для организации
- Sitemap.xml и robots.txt
- Open Graph и Twitter Card

### 10.4 Безопасность

- HTTPS повсеместно
- Content Security Policy (CSP)
- Web Application Firewall (WAF)
- Мониторинг целостности файлов
- Хранение секретов в переменных окружения
- Регулярные обновления зависимостей

## 12. Риски и митигация

| # | Риск | Вероятность | Влияние | Митигация |
|---|------|:-----------:|:-------:|-----------|
| 1 | Повторная компрометация сайта | Средняя | Критическое | WAF, мониторинг, автообновления, pen-testing |
| 2 | Низкая вовлечённость абитуриентов | Средняя | Высокое | Геймификация (Talapker), A/B тестирование, аналитика |

| 5 | Низкая производительность на слабых устройствах | Низкая | Среднее | Progressive enhancement, lazy loading, оптимизация бандла |
| 6 | Зависимость от сторонних API (Google Fonts, CDN) | Низкая | Низкое | Self-hosted шрифты, fallback на системные шрифты |
