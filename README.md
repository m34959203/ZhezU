# ZhezU — Новый сайт Жезказганского университета им. О.А. Байконурова

> Проект разработки нового сайта для **Жезказганского университета имени О.А. Байконурова** (https://zhezu.edu.kz/).
> Цель — создание современной, мультиязычной цифровой платформы, которая заменит текущий сайт университета.

---

## О проекте

**ZhezU** — это полноценное веб-приложение нового поколения для обновления официального сайта университета по адресу **https://zhezu.edu.kz/**. Проект объединяет приёмную комиссию, академический каталог, научный раздел, AI-центр, студенческую жизнь, карьерный центр и административную панель управления контентом в единой платформе.

### Текущее состояние (v3.0 — 27 февраля 2026)

| Метрика | Значение |
|---------|----------|
| Публичных страниц | **82** |

| API-эндпоинтов | **25** |

| Namespaces переводов | **22** |
| Ключей переводов | **~3 350+** на каждый язык |
| JSON-файлов данных (CMS) | **6** (news, settings, menu, homepage, university, contact) |
| Тестовых файлов | **8** (7 unit + 1 setup) |
| CI/CD Pipeline | **GitHub Actions** (lint, typecheck, test, build) |

### Целевая аудитория

- **Абитуриенты и родители** — информация о приёме, образовательных программах, подача заявок
- **Текущие студенты** — студенческая жизнь, общежития, спорт, клубы, события
- **Преподаватели и кураторы** — наука, публикации, конференции, лаборатории
- **Работодатели-партнёры** — пул талантов, профили студентов
- **Администрация университета** — CMS-панель, управление новостями, настройками, переводами, меню

### Ключевые возможности

- Мультиязычность: **қазақша**, **русский**, **English** (22 namespace-а, ~3 350+ ключей)
- Тёмная и светлая темы (с определением системных настроек)
- Адаптивный дизайн (мобильные устройства, планшеты, десктоп)
- Мега-меню с 6-ю разделами верхнего уровня
- SEO-оптимизация (hreflang, мета-теги, OG-теги, robots.txt, sitemap.xml)
- Админ-панель с JSON-based CMS (новости, настройки, переводы, меню, страницы, университет, главная)
- **Конструктор страниц (Page Builder)** — drag-and-drop управление блоками главной (10 типов: hero, программы, новости, кафедры, CTA, текст, изображение, баннер, HTML, разделитель)
- Аутентификация администратора (HMAC-SHA256 токены, httpOnly cookies)
- AI-функции: авто-перевод, анализ контента, валидация переводов (Google Gemini)
- Авто-публикация в соцсети (Telegram, Instagram)
- Загрузка фотографий (ректор, проректоры)
- Управление навигационным меню (добавление/удаление/перетаскивание)
- Публичные API для динамического контента
- CI/CD через GitHub Actions (lint, typecheck, test, build)

## Быстрый старт

### Требования

- Node.js 22+
- pnpm 10+

### Установка и запуск

```bash
# Клонирование репозитория
git clone <repository-url>
cd ZhezU/zhezu-app

# Установка зависимостей
pnpm install

# Запуск в режиме разработки
pnpm dev

# Сборка для продакшена
pnpm build

# Запуск продакшен-сервера
pnpm start
```

Приложение будет доступно по адресу `http://localhost:3000`.

### Переменные окружения

```bash
# Обязательные
ADMIN_SECRET=your-admin-password    # Пароль администратора (мин. 8 символов)

# Опциональные
PORT=3000                            # Порт сервера (Railway задаёт автоматически)
NEXT_PUBLIC_SITE_URL=https://...     # URL сайта для Railway
GEMINI_API_KEY=...                   # Ключ Google Gemini AI (для авто-перевода)
TELEGRAM_BOT_TOKEN=...               # Токен Telegram-бота (для авто-публикации)
TELEGRAM_CHAT_ID=...                 # ID чата Telegram
INSTAGRAM_ACCESS_TOKEN=...           # Токен Instagram (для авто-публикации)
```

### Админ-панель

- URL: `http://localhost:3000/admin`
- Пароль: задаётся через переменную окружения `ADMIN_SECRET`

### Скрипты

```bash
pnpm dev              # Запуск dev-сервера
pnpm build            # Production-сборка
pnpm start            # Production-сервер
pnpm lint             # ESLint проверка
pnpm format           # Prettier форматирование
pnpm format:check     # Prettier проверка
pnpm test             # Запуск тестов (watch)
pnpm test:ci          # Запуск тестов с покрытием
pnpm typecheck        # TypeScript проверка типов
```

### Деплой на Plesk (hoster.kz)

```bash
cd zhezu-app

# 1. Установить зависимости
pnpm install

# 2. Собрать билд и подготовить папку deploy/
bash scripts/deploy.sh

# 3. Закоммитить и запушить
cd ..
git add -A
git commit -m "build: описание изменений"
git push
```

> **ВАЖНО:** Папка `deploy/` содержит standalone-билд Next.js и коммитится в git. Plesk забирает изменения из git. Без шага `bash scripts/deploy.sh` изменения кода НЕ появятся на сайте!

### Docker

```bash
# Сборка образа
docker build -t zhezu-app .

# Запуск контейнера
docker run -p 3000:3000 -e ADMIN_SECRET=your-secret zhezu-app
```

## CI/CD Pipeline

Проект использует **GitHub Actions** для непрерывной интеграции:

```
Push/PR → Lint (ESLint + Prettier) → TypeScript → Unit Tests (Vitest) → Build (Next.js)
```

| Этап | Инструмент | Описание |
|------|-----------|----------|
| Lint | ESLint 9 + Prettier 3.8 | Проверка качества кода |
| TypeScript | tsc --noEmit | Проверка типов |
| Unit Tests | Vitest 4.0 + Coverage V8 | Тесты компонентов и утилит |
| Build | Next.js 16 | Production-сборка |
