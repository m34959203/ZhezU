## 3. Критические проблемы безопасности

### 3.1 Открытый API-ключ Google (.mcp.json)

**Серьёзность: КРИТИЧЕСКАЯ**

Файл `.mcp.json` содержит Google API-ключ в открытом виде:
```json
"X-Goog-Api-Key": "AQ.Ab8RN6LvGszeBtTdTS0tFP4g9l6LEznVrZaIu9gfEkYgz6_PTQ"
```

Этот ключ доступен всем, кто зайдёт на публичный GitHub-репозиторий. Удаление из текущей ветки недостаточно — ключ останется в истории коммитов (коммит `2472495`).

**Рекомендации:**
1. Немедленно отозвать/ротировать ключ в Google Cloud Console
2. Удалить ключ из истории Git (BFG Repo-Cleaner или `git filter-branch`)
3. Добавить `.mcp.json` в `.gitignore`
4. Использовать переменные окружения или секреты GitHub для хранения ключей

### 3.2 Отсутствие .gitignore

В репозитории нет файла `.gitignore`. Любые секреты, временные файлы или конфиденциальные данные могут быть случайно закоммичены.

### 3.3 CDN без Subresource Integrity (SRI)

Все файлы загружают Tailwind CSS через CDN без хеша целостности:
```html
<script src="https://cdn.tailwindcss.com?plugins=forms,container-queries"></script>
```

Если CDN будет скомпрометирован, злоумышленник сможет внедрить произвольный JavaScript на все страницы.

### 4.1 Полное отсутствие JavaScript

Ни один из 18 HTML-файлов не содержит функционального JavaScript. Все интерактивные элементы — мёртвые:

- Кнопки навигации, поиска, переключения языка
- Фильтры и сортировка в каталоге
- Пагинация
- Карусели новостей/вакансий
- Мобильное гамбургер-меню

- Текстовый редактор контента с панелью форматирования
- Переключение тёмной/светлой темы

### 4.2 Фрагментация дизайн-токенов

| home_page_1 | `#3B82F6` + `#1d56c9` | `#0A0E17` |
| home_page_2 | `#1d56c9` | `#111621` |
| student_portal_2 | `#3b82f6` | `#020617` |
| academics_catalog_1 | `#1d56c9` | — |
| academics_catalog_2 | `#D4AF37` | `#0b0e14` |
| admin_dashboard | `#eebd2b` | `#221d10` |
| career_center | `#eebd2b` | `#221d10` |

| content_editor | `#eebd2b` | `#221d10` |
| contact_page | `#eebd2b` | `#221d10` |

**Три "золотых" цвета:** `#FFD700`, `#eebd2b`, `#D4AF37` — используются как "primary" в разных файлах.
**Два "синих" цвета:** `#3B82F6` и `#1d56c9` — также несогласованы.

### 4.3 Массовое дублирование кода

- **home_page_1 vs home_page_2** — ~90% идентичны (разница только в теме)
- **admission_portal_1 vs admission_portal_2** — ~90% идентичны
- **footer_1 vs footer_2** — ~95% идентичны (разница — наличие блока рассылки)
- **academics_catalog_1 vs academics_catalog_2** — значительное дублирование
- Навигационный хедер повторяется с вариациями во всех полностраничных файлах
- Внутри файлов: карточки программ, вакансий, менторов — идентичные HTML-блоки повторяются 3-6 раз

Все это должно быть единым шаблоном с переключением тем и компонентной системой.

### 4.4 Дублирование импорта Material Symbols

В **9 из 10** проверенных файлов дублируется `<link>` для Material Symbols Outlined:
```html
<link href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined..." rel="stylesheet" />
<link href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined..." rel="stylesheet" />
```
Браузер загружает один и тот же шрифт дважды.

### 4.5 Захардкоженные hex-значения

Десятки hex-цветов используются напрямую в классах вместо дизайн-токенов:
```html
text-[#506795] bg-[#f8f9fb] border-[#e8ebf3] text-[#0e121b] bg-[#3a3422]
```
Эти значения повторяются сотни раз и не определены в Tailwind-конфигурации.

### 4.6 Неиспользуемые шрифты

Noto Sans импортируется и объявляется как `font-body` в Tailwind-конфиге нескольких файлов, но ни разу не применяется в разметке. Manrope используется только в одном файле (student_talent_profile).

## 5. Проблемы доступности (Accessibility / a11y)

### 5.1 Анти-паттерн `data-alt`

**Самая серьёзная проблема a11y в кодовой базе.** Все изображения реализованы как:
```html
<div class="bg-cover bg-center" style="background-image: url(...)" data-alt="Description"></div>
```

Атрибут `data-alt` — кастомный, не распознаётся:
- Скринридерами
- Поисковыми системами
- Браузерными фолбэками для изображений

Должно быть: `<img src="..." alt="Description">` или минимально `role="img" aria-label="Description"` на div.

### 5.2 Формы без связанных `<label>`

Во многих формах `<label>` существуют, но не связаны с `<input>` через `for`/`id`:
- admission_portal_1, admission_portal_2
- contact_feedback_page
- career_center_job_board
- academics_catalog_2

### 5.3 Кнопки без `aria-label`

- Гамбургер-меню (все файлы)
- Кнопки навигации карусели
- Кнопки уведомлений
- Кнопки зума на карте
- Кнопки-иконки без текста

### 5.4 Отсутствие `aria-current="page"`

Ни в одном файле активный пункт навигации не помечен `aria-current="page"`.

### 5.5 Графики/диаграммы без альтернативного текста

- SVG-графики GPA (student_portal_2, student_portal_3) — нет `aria-label` или `<title>`
- CSS-гистограмма в career_center_dashboard — полностью непрочитываема скринридером
- Радарная диаграмма в student_talent_profile — нет текстовой альтернативы

### 5.6 Отсутствие skip-links, focus management, WCAG 2.1 AA

- Нет skip-ссылок для перехода к основному контенту
- Нет управления фокусом для модальных окон/меню
- Нет `prefers-reduced-motion` для анимаций (animate-ping в mentorship)
- `<html lang="en">` захардкожен — нет поддержки казахского/русского языков

## 6. Проблемы адаптивности (Responsive Design)

### 6.1 Боковая панель не сворачивается

В файлах с app-shell layout (`student_portal_2`, `student_portal_3`, `career_super-service_hub`, `content_editor_workspace`) сайдбар всегда видим (256px `w-64`). На мобильных устройствах он занимает всю ширину экрана, делая контент недоступным.

### 6.2 Нефункциональное мобильное меню

Кнопка-гамбургер присутствует во всех полностраничных файлах, но нигде нет JavaScript для отображения мобильной навигации.

### 6.3 Фиксированный viewport без прокрутки

Файлы с `overflow-hidden h-screen` на `<body>` создают фиксированный layout, который может не работать на устройствах с нестандартной высотой экрана или виртуальной клавиатурой.

## 7. Проблемы производительности

| Проблема | Влияние |
|----------|---------|
| Tailwind CSS CDN (~3 МБ JS-рантайм) | Крайне медленная загрузка, JIT-компиляция в браузере |
| Все изображения через CSS `background-image` | Невозможен `loading="lazy"`, нет `srcset` для responsive |
| Нет минификации HTML/CSS | Увеличенный размер файлов |
| Нет оптимизации PNG-скриншотов (~300 КБ каждый) | 29 × 300 КБ = ~8.7 МБ только на скриншотах |
| Дублированные загрузки шрифтов | Лишние сетевые запросы |
| Неиспользуемый шрифт Noto Sans | Бесполезный сетевой запрос |
| `blur-3xl` эффекты в footer | Высокая GPU-нагрузка на слабых устройствах |
| `prose` класс без подключённого плагина Typography | В content_editor — стили Prose не применяются |

## 13. Заключение

Репозиторий ZhezU представляет собой **коллекцию UI-прототипов** для редизайна сайта Жезказганского университета, созданных с помощью Google Stitch. Проект находится на ранней стадии — это визуальные макеты без функционала, backend-логики или инфраструктуры сборки.

**Сильные стороны:**
- Широкий охват: 30 различных экранов/компонентов
- Качественный документ анализа дизайна (DESIGN_ANALYSIS.md)
- Современные технологии (Tailwind CSS, Material Design)
- Адаптивная вёрстка с breakpoints

**Ключевые проблемы:**
- Открытый API-ключ в публичном репозитории
- Нулевая интерактивность (0 строк функционального JS)
- Фрагментированная дизайн-система (4+ версии "primary" цвета)
- Массовое дублирование кода (~90% между вариантами одной страницы)
- Серьёзные проблемы доступности (`data-alt`, отсутствие aria-атрибутов)
- Значительные расхождения между анализом/рекомендациями и реализацией
- Полностью отсутствующая инфраструктура (нет .gitignore, package.json, CI/CD)

Для перехода от прототипов к рабочему сайту потребуется: единая дизайн-система, компонентный фреймворк, CMS-интеграция, и значительная работа по доступности и производительности.
